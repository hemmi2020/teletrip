Booking Workflow

Standard HBX Group workflow. Availability called once, CheckRate only for RECHECK, rateKey passed unmodified.

Step 1
Availability
/hotel-api/1.0/hotels
→
Step 2 (conditional)
CheckRate
/hotel-api/1.0/checkrates
→
Step 3
Booking
/hotel-api/1.0/bookings

CheckRate called only when rateType = "RECHECK". Skipped entirely for "BOOKABLE" rates.

RequirementStatusDetails
2.1 Correct workflow order PassAvail → CheckRate (if RECHECK) → Booking. No repeated calls.
2.2 No repeated Availability PassNever re-called before CheckRate or Booking.
2.3 Max hotels per call PassUp to 2,000 hotels per Availability call.
2.4 All rooms in same call PassMulti-room via multiple occupancy nodes.
2.5 CheckRate only for RECHECK PassBOOKABLE rates skip CheckRate.
2.6 Batch CheckRate PassUp to 10 rates grouped per call.
2.7 Display promotions PassPromotions shown before booking.
